# 2026-03-25 Post-H50 Origin Mechanism Reentry Master Plan

## Summary

`H50` remains a correct negative closeout for the post-`H49` widened
internal-route question. It did not show bounded system value over simpler
baselines on the landed `R51` rows, and it should not be overturned by
momentum.

This plan therefore does not reopen `F27`, does not reactivate transformed or
trainable executor entry, and does not treat "LLMs are computers" as an
acceptance target.

Instead, it saves one new explicit mechanism-first lane anchored to a narrower
reading of the local-only Origin materials: the object to reproduce or falsify
is whether an append-only execution trace plus exact structured retrieval can
support a bounded free-running internal executor on a transparent reference
contract.

The fixed mainline order is:

`F28_post_h50_origin_mechanism_reentry_bundle` ->
`H51_post_h50_origin_mechanism_reentry_packet` ->
`R55_origin_2d_hardmax_retrieval_equivalence_gate` ->
`R56_origin_append_only_trace_vm_semantics_gate` ->
`R57_origin_accelerated_trace_vm_comparator_gate` ->
`H52_post_r55_r56_r57_origin_mechanism_decision_packet`.

The fixed low-priority sidecar is:

`P37_post_h50_narrow_executor_closeout_sync`.

## Scientific Target

This wave narrows the current scientific target to one explicit mechanism
chain:

1. deterministic execution can be represented as an append-only execution
   trace;
2. latest relevant state can be recovered exactly from that trace by a
   transparent reference lookup;
3. the claimed `2D` hard-max retrieval path can match that reference exactly;
4. those primitives can support one bounded free-running trace-VM semantics
   contract exactly; and
5. only after that exact chain is in place may any fast-path comparator claim
   be evaluated.

The strongest claims remain blocked:

- arbitrary `C`;
- broad Wasm coverage;
- general "LLMs are computers";
- transformed-model entry;
- trainable-executor entry; and
- merge back to `main`.

## Chosen Route

The chosen route is:

- `F28` saves the narrowed mechanism claim delta and fixes the exact lane
  order;
- `H51` preserves negative `H50` while authorizing one new explicit
  mechanism-first route;
- `R55` isolates whether the claimed `2D` hard-max retrieval path is exactly
  equivalent to a transparent reference lookup;
- `R56` isolates whether append-only trace plus exact retrieval support one
  bounded free-running trace VM exactly;
- `R57` evaluates whether the accelerated trace-VM path retains bounded value
  relative to transparent reference execution on the exact `R56` rows; and
- `H52` closes the lane explicitly without widening the claim ceiling.

`P37` runs in parallel as an operational/docs sync packet for worktree
discipline, artifact slimming, and branch hygiene during this wave.

## F28 Contract

`F28` must remain planning-only and must do five things:

- preserve `H50` as the preserved prior docs-only closeout and preserve `H43`
  as the current paper-grade endpoint;
- rewrite the active claim ladder around the narrower mechanism chain rather
  than around transformed or trainable entry;
- fix `H51` as the only follow-up packet for this reentry;
- fix `R55 -> R56 -> R57 -> H52` as the only admissible execution order; and
- keep `F27`, `R53`, and `R54` explicitly blocked.

## P37 Contract

`P37` remains operational/docs-only and must codify:

- the clean `F28/H51` worktree as the control surface for this wave;
- descendant clean worktrees as the only scientific execution surfaces for
  `R55` and `R56`;
- raw step rows, per-read rows, and any artifact above roughly `10 MiB` as
  out-of-git by default;
- a separate commit cadence for `F28/H51/P37`, `R55`, `R56`, `R57`, and
  `H52`; and
- continued no-merge posture for dirty root `main`.

## H51 Contract

`H51` is docs-only and must make one narrow decision:

- `authorize_origin_mechanism_reentry_through_r55_first`;
- `keep_h50_stop_state_as_terminal`; or
- `reactivate_f27_trainable_or_transformed_entry`.

The only admissible positive outcome here is
`authorize_origin_mechanism_reentry_through_r55_first`.

That outcome must preserve negative `H50` as a bounded-value falsifier on the
broader post-`H49` route while still recognizing that the narrower Origin
mechanism chain has not yet been tested directly at the required retrieval,
trace-semantics, and fast-path-equivalence levels.

## R55 Contract

`R55` is the only next runtime candidate fixed here. It must test whether the
claimed `2D` hard-max retrieval path is exactly equivalent to a transparent
reference latest-relevant-state lookup on a fixed bounded suite.

Required outputs:

- exact value parity;
- exact maximizer-row parity, including declared tie handling;
- duplicate-max and overwrite-after-gap coverage;
- first-fail localization; and
- one explicit lane verdict.

If `R55` fails, `R56` and `R57` do not open.

## R56 Contract

`R56` runs only after a positive exact `R55`. Its question is whether
append-only trace plus exact retrieval support one bounded free-running trace
VM semantics contract without hidden mutable side channels or external
execution at tested runtime.

Required outputs:

- full step-trace parity;
- final-state parity;
- declared instruction-level transition logs;
- first-fail localization; and
- one explicit lane verdict.

Teacher forcing, answer-only agreement, or compiler-side replay do not count
as success.

## R57 Contract

`R57` runs only after a positive exact `R56`. Its question is not whether the
mechanism merely exists, but whether the accelerated trace-VM path retains
bounded value relative to transparent reference execution on the exact `R56`
rows.

Required comparators:

- accelerated internal trace-VM execution;
- reference linear trace-VM execution; and
- transparent external interpreter execution where the comparison remains
  meaningful.

Required outputs:

- exactness parity across comparators where applicable;
- end-to-end latency;
- retrieval-share decomposition;
- trace-length sensitivity; and
- one explicit lane verdict.

## H52 Decision Rule

`H52` is docs-only and must read `R55`, `R56`, and `R57` together.

Allowed outcomes:

- `freeze_origin_mechanism_supported_with_fastpath_value`;
- `freeze_origin_mechanism_supported_without_fastpath_value`; or
- `stop_as_partial_mechanism_only`.

`H52` must preserve `H43` as the paper-grade endpoint unless a later explicit
packet raises the claim ceiling on stronger evidence than this wave.

## Defaults

- Do not reopen `F27`, `R53`, or `R54`.
- Do not widen into arbitrary `C`, broad Wasm, or demo-first presentation.
- Do not count external execution during tested runtime as internal execution
  evidence.
- Prefer exact equivalence and clear falsifiers over larger demos.
- Keep dirty root `main` unmerged and out of scope for scientific execution.
