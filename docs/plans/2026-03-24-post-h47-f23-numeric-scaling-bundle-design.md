# Post-H47 F23 Numeric Scaling Bundle Design

## Objective

`F23_post_h47_numeric_scaling_bundle` is the only admissible planning bundle
between landed `P35` and any new post-`H47` runtime execution.

The bundle stays deliberately narrow:

- `H47` remains the active docs-only packet;
- `H43` remains the paper-grade endpoint;
- exact `R46/R47` evidence remains decisive;
- comparator-only `R48` remains non-substitutive; and
- `F24` remains dormant.

## Locked Inputs

- current active docs-only packet:
  `H47_post_r48_useful_case_bridge_refreeze`
- current paper-grade endpoint:
  `H43_post_r44_useful_case_refreeze`
- preserved prior exact runtime gate:
  `R46_origin_useful_case_surface_generalization_gate`
- current exact frontend bridge gate:
  `R47_origin_restricted_frontend_translation_gate`
- current comparator-only useful-case model gate:
  `R48_origin_dual_mode_useful_case_model_gate`
- current exact-first planning bundle:
  `F21_post_h43_exact_useful_case_expansion_bundle`
- current comparator-planning bundle:
  `F22_post_r46_useful_case_model_bridge_bundle`

## Bundle Outputs

`F23` must save exactly four planning tables:

- `precision_regime_matrix`
- `addressability_strategy_table`
- `length_bucket_plan`
- `kill_criteria`

It must also fix:

- `R49_origin_useful_case_numeric_scaling_gate` as the only next runtime
  candidate;
- `F25_post_h48_restricted_tinyc_lowering_bundle` as the positive downstream
  placeholder; and
- `P36_post_h48_falsification_closeout_bundle` as the negative downstream
  placeholder.

## Acceptance

- the bundle remains planning-only;
- `H47` remains current and `H43` remains the paper-grade endpoint;
- `R49` is the only next runtime candidate;
- no broader frontend/runtime scope lift is authorized;
- `F24` remains dormant; and
- `F25/P36` remain placeholders only until later explicit packets.
