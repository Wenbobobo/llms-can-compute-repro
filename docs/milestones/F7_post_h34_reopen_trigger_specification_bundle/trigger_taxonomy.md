# Trigger Taxonomy

`F7` does not search for a reopen candidate. It specifies the only trigger
classes that could ever justify a later explicit same-substrate contradiction
packet, and it keeps all other motivations out of bounds.

## Admissible trigger families in principle

| Trigger family | Why it is admissible in principle | Minimum shape of a future packet |
| --- | --- | --- |
| Exactness or final-state contradiction on the current admitted row or named same-family boundary probe | would directly contradict the current `R37/R38/R39/H34` support line | one concrete failing row, one exact comparator set, one predeclared success/failure rule |
| Sharper same-row control-surface dependence than the declared `R39` helper-body permutation | would test whether the current runtime claim still depends on undeclared control-shaping choices | one named perturbation family, one fixed baseline/variant comparator set, one stop rule |
| Sharper evidence that compiler/helper encoding is carrying the substantive work while runtime contribution becomes irrelevant on the same rows | would target the remaining claim-relevant "compiler did all the work" objection without changing substrate | one declared comparator isolating runtime relevance, one non-drifting row set, one fixed contradiction criterion |

## Inadmissible trigger families from the current state

| Trigger family | Why it is inadmissible now |
| --- | --- |
| A wish to try another compiled family, broader opcode surface, Wasm-like support, or arbitrary `C` | this is scope lift, not contradiction inside the current supported boundary |
| Historical same-endpoint systems disappointment | this belongs to the preserved `H21/H27` route, not the active Origin-core line |
| Frontier pressure, demo pressure, or origin-headline dissatisfaction | these are narrative concerns, not contradiction packets |
| General suspicion without one unique predeclared comparator | `F5` already rejects reopen-by-vagueness |

## Routing consequence

Until one admissible family is instantiated as exactly one concrete candidate,
the current state remains unchanged:

- `H32` stays active;
- `H34` stays current;
- `F5` stays concluded at `no_reopen_candidate_survives`;
- no `H35` should be created by default;
- no active downstream runtime lane exists.
