# Scientific Target Ladder

This ladder is a planning aid only. It orders targets from the currently
supported Origin-core line outward to families that remain blocked or need a
new substrate. It does not activate any lane.

| Ladder rung | Scientific target | Current status | Why |
| --- | --- | --- | --- |
| 1 | Append-only trace plus exact retrieval plus a small exact executor on the current Origin-core substrate | `supported_here` | `H28 -> R34/R35 -> H29` preserves the narrow substrate claim as the current scientific floor. |
| 2 | One tiny compiled family plus one richer same-opcode control/call family on the same substrate | `supported_here` | `R37/R38/R39/H32/H34` preserve narrow compiled-boundary support without widening it into a broad compiler claim. |
| 3 | Same-substrate contradiction scouting and future gating around the current admitted row and named boundary probe | `planning_only` | `F5/F6` keep future decisions docs-only and do not authorize execution. |
| 4 | Beyond-Origin bridge framing that explains which future family would count as a true scope change | `planning_only` | `F8` can classify future bridge surfaces without selecting a runtime lane. |
| 5 | Restricted semantic-boundary frontend beyond the current tiny bytecode line | `blocked_by_scope` | Wasm-like or richer compiled semantics are currently blocked until a later explicit scope decision plus stronger evidence replaces the standing no-widening state. |
| 6 | Executor-value comparator work for richer semantic obligations before any frontend widening | `planning_only` | A comparator matrix can stay docs-only, but it is not an active implementation or runtime family under `H34`. |
| 7 | Hybrid planner-plus-executor bridge where a broader planner hands work to the exact executor substrate | `requires_new_substrate` | This would move beyond the current narrow executor story into a hybrid system family that the repo does not yet support. |
| 8 | General LLM-computer, arbitrary `C`, or broad systems-superiority story | `requires_new_substrate` | Current evidence is intentionally too narrow and current packets explicitly reject headline scope lift. |

Rungs 5 through 8 are not active lanes. They remain downstream of a later
explicit packet and must not be inferred from the current `H34` freeze.
