# Reopen Admissibility Rules

Any future same-substrate reopen packet after `H34` must name all of the
following in advance:

1. one concrete contradiction, failure mode, or sharper unresolved dependency
   tied directly to the current admitted row and/or the named same-family
   boundary probe;
2. one predeclared same-substrate comparator set;
3. no new opcode surface, no hidden host evaluator, and no family-breadth
   drift;
4. one fixed success/failure criterion chosen before execution;
5. one explicit stop condition that prevents an open-ended repair loop.

The following do not satisfy admissibility by themselves:

- dissatisfaction with the marketing headline in the origin post;
- desire for broader Wasm-like or arbitrary-`C` coverage;
- preserved same-endpoint systems negativity;
- broad suspicion that "the compiler did all the work" without one sharper,
  predeclared same-substrate test;
- frontier/demo pressure or publication convenience.

## Current application

Applied to the landed `R37/H30/H31/R38/H32/H33/R39/H34` bundle, these rules
yield `no_reopen_candidate_survives`.

That means:

- `H32` remains the active routing packet;

- `R39` remains completed downstream evidence, not a routing change;
- no new `H35`-style packet should be created by default;
- any future same-substrate reopen must begin with a new explicit contradiction
