# Stop Conditions

Stop the future audit immediately if any candidate:

- changes source meaning instead of stressing runtime relevance;
- requires a new opcode or richer value family;
- requires a new substrate or external runtime comparator;
- cannot keep the admitted row plus named boundary row as the only rows;
- cannot predeclare the slice pair mechanically on the same row.

Operational stop rules:

- stop after the first uniquely isolated contradiction;
- stop if all surviving candidates are still non-unique;
- stop if the evidence packet starts drifting toward broader compiler or
  frontier rhetoric.
